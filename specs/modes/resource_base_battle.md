# 資源拠点争奪戦

> 元仕様: <https://wiki.strugarden.pluslake.net/対人/資源拠点争奪戦>

## 概要

**国 (陣営) 単位で拠点の支配権を賭けて行う大規模PvPモード**。5人パーティで特定拠点に参加し、個々の勝敗が拠点単位の支配権に反映される。

本再実装ではフェーズ3コンテンツとして位置付け、MVPには含めない。ただしデータモデルレベルでは最初から拡張を想定する。

## 参加条件

- レベル15以上のキャラクター
- 5人パーティの組成が必須
- 陣営 (国) に所属していること
- **参加可能時間帯: 20:00 〜 翌 6:00** (元仕様準拠、「八百長対策として制限」)

## マップ

全17種類の拠点マップ:

- **主要拠点**: リングランド草原、ブルーム平原、ルセデス山地、キュリル半島、ラヴィア半島、ルデンヌ大森林、ロナデ高原、ルカン平野、マリス平原、シュロ陸峡、ファンメル山地、ソーン半島、イレネイド山脈、グレフ雪原、ナビア北限地帯、ルルド台地
- **特殊マップ**: 縁(鳥居)青、縁(鳥居)赤

再実装では物量があるため、優先順位を決めて段階的に実装する (Phase 3 で主要5種 → Phase 4 で全17種)。

## 対戦構造

- 1試合は **5人 vs 5人** の対戦。
- 拠点ごとに同時に複数の試合が進行している (マッチ並列性あり)。
- 各拠点には「支配国」が設定されている。挑戦側と防衛側に分かれる。

## 勝敗による支配権変動

- 個々のマッチ結果が **拠点ポイント** に加算される。
- 拠点ポイントが規定値を超えた国が支配権を得る。
- **支配権更新タイミング**: 水曜日の定期メンテナンス (週次)。

## 功績ポイント (個人報酬)

| イベント | 効果 |
|---------|-----|
| 勝利 | 功績ランクと同数の功績ポイントを獲得 |
| 敗北 | 功績ランク -1 |
| 午前6時 | 功績ランクが 10 に戻る (日次リセット) |
| 功績ランクが 0 | 参加不可 (翌6時まで) |

- 功績ランクはレベル10からスタート (日次上限)。
- 功績ポイントは定期的に報酬アイテム (`争奪褒賞品` 系) に交換可能。

再実装ではこの仕組みをそのまま踏襲し、以下のようにモデル化する:

```
PlayerConquestStats {
  player_id
  current_rank   (1-10, 日次でリセット)
  lifetime_points
  last_reset_at
}
```

## ギルド報酬

- 勝利した個人の **所属ギルドにギルド功績点** が付与される。
- ギルドランキングに反映。

## 戦況表示

運営画面/ゲーム内ロビーで、以下のような戦況表示を行う (元仕様準拠):

- 「情勢は平穏で戦火なし」 — 累計ポイントが低い状態
- 「◯◯◯◯ がかなり優勢」 — 特定国が優位
- 「混戦状態」 — 複数国が競っている状態

ポイント閾値を設定し、表示文言を切り替える。

## 実装上の注意

- **時間帯制限**: サーバタイムで 20:00〜翌6:00 のみキュー可能。
- **反作為 (八百長対策)**: 同一プレイヤー間の連続マッチを避ける、短時間の敗退連鎖を検出する等。
- **大規模同時接続**: 拠点ごとに試合並列実行されるためバックエンドのスケールアウトを前提とする。
- **週次処理**: 水曜メンテ時のバッチ処理で拠点支配権を確定 → アナウンス。

## データフロー

- `ConquestSeason` (争奪戦シーズン)
- `BasePoint` (各拠点の支配状態とポイント)
- `ConquestBattleQueue` (キュー中パーティ)
- `ConquestBattleLog` (試合単位の結果 + ポイント加算)
- `PlayerConquestStats` (個人功績ランク)
- `GuildConquestStats` (ギルド功績点)
- `Faction` (国/陣営の定義とメンバー)
- `MatchRecord` (共通)
