# 闘技場

> 元仕様: <https://wiki.strugarden.pluslake.net/対人/闘技場>

## 概要

ランキング競争を伴う、**公式マッチメイキング型のPvPモード**。デュエルと異なり、ルールは固定。運営 (ゲーム側) が定めたルールで公平に競う。

闘技場には3つのサブモードがある:

1. **デュエル闘技場** — 個人ランキングを競う (1vN、デッキ制限有り)
2. **ギルド闘技場** — ギルド単位でランキングを競う
3. **闘錬の間** — 対NPC戦 (詳細は [`training_hall.md`](./training_hall.md))

本ファイルでは 1, 2 をカバーする。闘錬の間は別ファイル。

## 共通ルール

| 項目 | 値 |
|------|----|
| 移動フェイズ時間 | 30秒 |
| 行動フェイズ時間 | 30秒 |
| マップ | 闘技場専用マップ (10種類予定、ランダム抽選) |
| 勝利条件 | 敵が2人以下の場合: 敵を全滅 / 敵が3人以上: リーダーを倒せ |
| ターン制限 | 30ターン (タイムアップ時は規定ルールで勝者決定) |
| タイムアウト時挙動 | 入力なし = 待機 |

## デュエル闘技場 (個人戦)

### 参加条件

- 自キャラクターのレベル範囲でマッチメイキング。
- 使用可能デッキに **制限あり** — 特定のデッキ (初心者デッキ、旅人デッキ、傭兵デッキ 等) にのみ限定する場合がある。

### チーム編成

- 1v1 または 1vN のマッチ。
- ソロキュー想定 (Phase 2 以降でパーティキューも対応)。

### マッチメイキング

- MMR (Match Making Rating) をプレイヤー単位で管理。
- ELO方式または Glicko 方式をベースに、勝敗でレーティングが上下する。
- マッチング時は `|MMR差| < threshold` かつ `待機時間` に応じてしきい値を緩和する。
- 同一キャラクター (重複プレイ防止のためアカウントID) が同時にキューに入らない。

### ランキング

- 週次・月次・シーズン単位のランキングを公開。
- シーズン終了時にランク報酬 (装飾品、アイコン、デッキ権利書) を配布。
- 不正検出 (ブースト試合、放置試合、長時間の引き分け繰り返し) をチェック。

### リーダー制 (3人以上チーム対戦時)

- 参加プレイヤーの中から1人を「リーダー」に指定 (入室時/部屋作成時)。
- リーダーが戦闘不能になった時点で即時敗北。
- リーダーHP/AP はUIで特殊表示する。

## ギルド闘技場

### 参加条件

- ギルド所属が必須。
- 一定ランク以上のギルドのみエントリー可能 (Phase 3)。

### 対戦構造

- 5v5 (暫定) のギルド対抗戦。
- 対戦相手はギルドランキングから抽選 or チャレンジ制。

### ランキング

- ギルド単位で累積ギルドポイントを管理。
- 週次でランキングスナップショットを取り、上位ギルドに報酬。

### 集計

- 個人貢献 (撃破数、被撃破数、フラッグ貢献) を集計し、ギルド内のMVPを抽出。

## 使用可能アイテム/スキル

- 闘技場では **回復アイテム使用不可** (アイテム封印ルール、Phase 2で要検証)。
- `ブレイク系` スキルのうち運要素の高すぎるもの (オールオアナッシング等) は要素の性質上制限しない方針だが、バランス調整で最終決定する。

## マッチフロー

```
1. プレイヤーがキュー登録 (使用デッキ/キャラクタースナップショット送信)
2. Matchmaker が相手を決定
3. 全参加者にマッチ確定通知
4. マッチサーバにセッション作成
5. 戦闘開始 → ターン進行 → 勝敗確定
6. MatchRecord 永続化 → MMR/ランキング更新 → 報酬付与
```

## 実装上の注意

- **一貫性**: 同じ条件のマッチでも結果が再現可能 (決定論的シミュレーションを利用)。
- **AFK/切断対策**: タイムアウトで連続 N 回入力なし → 自動投了処理。
- **観戦**: トップランカーの試合は観戦可能 (Phase 2 で追加)。
- **リプレイ公開**: 試合終了後、自身の試合リプレイのみダウンロード可能。上位試合は公開設定可能。

## データフロー

- `ArenaSeason` (シーズン定義)
- `PlayerArenaStats` (MMR、勝率、シーズン成績)
- `GuildArenaStats` (ギルド単位)
- `ArenaQueueTicket` (キュー中のプレイヤー)
- `MatchRecord` (試合結果、共通)
