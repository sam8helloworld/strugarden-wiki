# 実装ロードマップ

MVP → フル実装までのフェーズ分けと、各フェーズで達成すべきマイルストーン。

## Phase 0: 準備 (リサーチ & スケルトン)

**目的**: チームで共通認識を作り、基盤コードを最小限並べる。

- [ ] 仕様書 (`specs/`) レビュー
- [ ] 未移植ページ (デュエル/闘技場/闘錬の間/資源拠点争奪戦/経験値) の docs 側マークダウン化
- [ ] マスターデータの雛形 (YAML/JSON) をリポジトリに用意
  - `masters/jobs.yaml`, `masters/skills.yaml`, `masters/states.yaml`, `masters/maps.yaml`, `masters/equipment.yaml`
- [ ] ソース構成の決定 (エンジン/サーバ/クライアントのディレクトリ)
- [x] 決定論 PRNG 実装の選定 (SplitMix64 等)

**完了条件**: 仕様書が整合し、ディレクトリ構造とマスタースキーマが確定している。

## Phase 1: MVP (対人1v1が動く)

**目的**: 最小限の対人戦を体験できる状態。デュエル(1v1)と闘錬の間 (対AI) を提供する。

### 1.1 戦闘エンジン (最小セット)

- [x] `BattleState` / `BattleUnit` / `Map` の実装
- [x] 移動フェイズ (段差・通行判定)
- [x] 行動フェイズ (スピード順解決)
- [x] スキル `ap_cost` / `range_shape` / `effect_shape` の適用
- [x] 独自ダメージ計算式 ([`battle_rules.md §4.5`](./battle_rules.md#45-ダメージ計算) に準拠、背後判定・致命打・±5%変動・属性相性を含む)
- [x] 状態異常 核となる数種類: 麻痺 / 睡眠 / 暗闇 / 毒 / 炎上 / 凍結 / 防御 / 見切り / 戦闘不能
- [x] KO・勝敗判定
- [ ] イベント列出力・リプレイ再生

### 1.2 マスターデータ (全ジョブ対応)

- [x] **基本ジョブ 6種**: 戦士・格闘士・盗賊・守護魔導師・精霊魔導師・黒印魔導師
- [ ] **上位ジョブ 6種**: 騎士・幻闘士・忍者・次元天導師・召喚天導師・錬金天導師
- [ ] 上位職解放条件 (Lv45 + ドラゴニアメインクエスト第1話相当) を、再実装では「Lv上限付きのアンロック条件」に置換
- [ ] サイドジョブ選択 (戦士↔盗賊↔格闘士↔守護↔精霊↔黒印 のリング構造) のデータモデル対応
- [ ] 全12ジョブのCMDスキルツリー + PSVスキル一覧 を [`../docs/jobs/`](../docs/jobs/) から取り込み
- [ ] 各ジョブ 40〜60 スキル、合計 **500〜700 スキル** 相当を `masters/skills.yaml` に登録
- [ ] 武器: 全9種 (片手剣・両手剣・片手斧・両手斧・棍棒・刀・槍・ポールウェポン・短剣・杖・弓・銃・ボウガン)
- [ ] 防具: 上下鎧・兜・篭手・ブーツ・盾 (各3段階の重量)
- [ ] 装飾品: 6部位 (耳/顔/指/首/上装飾/下装飾)
- [ ] デッキ: 少なくとも最大枠12のデッキまで全シート対応 (初心者〜ドラグーン、特殊デッキ)
- [ ] マップ: デュエル用5種、闘錬の間用 3〜5種

### 1.3 サーバ・クライアント基盤

- [ ] Auth API (signup / login)
- [ ] Character API (作成 / 装備変更 / デッキ編集)
- [ ] Battle Realtime (デュエル1v1のみ)
- [ ] Matchmaking (デュエル: ルームベース + 闘錬の間: 即開始)
- [ ] Replay API (取得・再生)

### 1.4 闘錬の間 (AI対戦)

- [x] シンプルなヒューリスティックAI (最も近い敵に攻撃)
- [ ] 3ランク (戦豹 / 猛虎 / 荒獅子)
- [ ] ランクごとの固定AIデッキ
- [ ] 全12ジョブでソロプレイ可能なバランス調整 (各ジョブ × 3ランクを最低1プリセット用意)

### 1.5 UI

- [x] マップレンダリング (2D、スプライトorタイル)
- [x] ユニット移動・行動の可視化
- [ ] スキル発動アニメーション (簡易)
- [x] デッキ編集 UI

**完了条件**: 2人のプレイヤーが1v1デュエルを完遂でき、リプレイが保存・再生できる。ソロプレイヤーは闘錬の間で2段階までクリアできる。**全12ジョブ** がデュエル・闘錬の間の両方で選択・プレイ可能で、各ジョブのユニークスキル (騎士の陣、幻闘士の石化・時限爆発、忍者の回復無効バリア、天導師の上位魔法など) が意図通り動作する。

## 優先度順の依存関係

```
Phase 0 ───► Phase 1 ───► Phase 2 ───► Phase 3 ───► Phase 4
 (準備)      (MVP)         (本格)        (大規模)      (磨き)

            ↑                  ↑
            └── バランス調整  ──┘
                (継続的)
```

Phase 1 を越えた時点で **公開プレイテスト** を実施し、以降はプレイヤーフィードバックでフェーズ2の優先度を調整する。

## Phase 1 完了の目安 KPI

- 30分以内に新規プレイヤーが最初のデュエルを完遂できる
- 試合の決定論テストが 100%成功する (乱数ずれなし)
- 代表10種のスキルコンボが意図通り動く (手動確認 + Goldenテスト)
- サーバ1プロセスで並列50試合を処理できる
